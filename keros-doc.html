Keros API documentation version v1

/api/v1

This API represents the interface between Keros-Back and Keros-Front. All requests and responses are of JSON type.

/auth

Authentication endpoints

/auth/login post

post

Allows the user to log into the platform

post /auth/login

Allows the user to log into the platform

  • Request
  • Response

Body

Media type: application/json

Type: object

Properties
  • username: required (string)
  • password: required (string)

    Should be encrypted already

Example:

{
  "username": "thomasD",
  "password": "foobar"
}

HTTP status code 200

Returned when authentication is successful

Body

Media type: application/json

Type: object

Properties
  • token: required (string)

Example:

{
  "token": "thomasD"
}

HTTP status code 401

/core

Core endpoints, used at multiple places in the application

/core/member get post

get

Gets a page of CoreTypes.Member. The response is under the form { "content": [ITEMS], "meta": {META} }

post

Creates an CoreTypes.Member and returns it

get /core/member

Gets a page of CoreTypes.Member. The response is under the form { "content": [ITEMS], "meta": {META} }

Secured by OAuth2-0

Simplified OAuth2.0 authentication in which every request must containt a JWT token header

  • Request
  • Response
  • Security

Query Parameters

  • search: (string)

    Allows a search where any information of each item must match the searc term

    Example:

    thomas
  • orderBy: (string)

    The field by which to order by

    Example:

    name
  • order: (one of desc, asc - default: asc)

    Order

  • pageNumber: (integer - default: 0)

    The number of the page that is needed

  • pageSize: (integer - default: 25 - minimum: 10 - maximum: 100)

    Number of elements per page to recover

HTTP status code 200

Body

Media type: application/json

Type: object

Properties
  • content: required (array of object)

    Items: Member

    • id: required (number)
    • username: required (string)
    • firstName: required (string)
    • lastName: required (string)
    • genderId: required (number)
    • email: required (string)
    • birthday: required (date-only)

      No time, only a date of format YYYY-MM-DD

    • departmentId: (number)

      the ID of the member's school department

    • schoolYear: (one of 1, 2, 3, 4, 5, 6, 7, 8)

      A INSA Lyon school year

    • telephone: (string)
    • addressId: required (number)
    • positionIds: required (array of number)

      The ID of the member's position(s) within ETIC INSA Technologies. Can have from 1 to N items.

    Example:

    {
  "id": 5,
  "username": "jconnor",
  "firstName": "John",
  "lastName": "Connor",
  "genderId": 5,
  "email": "john.conor@terminator.com",
  "birthday": "1967-01-20",
  "departmentId": 5,
  "schoolYear": 3,
  "telephone": "0033675489521",
  "addressId": 7,
  "positionIds": [
    4,
    6
  ]
}
  • meta: required (object)

    The page metadata. Note that when there are 2 pages, it means that the pages 0 and 1 exist, but not 2 (i.e. the page number is an index).

    • page: required (integer)
    • totalPages: required (integer)
    • totalItems: required (integer)
    • itemsPerPage: required (integer)

    Example:

    {
  "page": 0,
  "totalPages": 2,
  "totalItems": 40,
  "itemsPerPage": 25
}
